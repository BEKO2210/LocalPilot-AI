# Sales-Playbook – LocalPilot AI

Konkreter Verkaufsleitfaden. Adressiert: Erstkontakt → Pitch
→ Vertrag → Onboarding (7 Tage) → Live → Übergabe → Support.

> **Stand:** Session 85 (Phase 3 startet). Erste Pilotkunden-
> Welle anvisiert für Sessions 88–93. **Bei Session 100 muss
> jeder Schritt hier ohne Code-Änderung abrufbar sein.**

---

## 1) Zielgruppe (3 Hero-Branchen, ab Session 91 fokussiert)

Wir verkaufen **nicht** an alle 13 Branchen-Presets gleich gut.
Pilotwelle adressiert nur diese drei:

| Branche               | Warum                                           | Preset-Key            |
| --------------------- | ----------------------------------------------- | --------------------- |
| **Friseur / Barber**  | Hohe Foto-Affinität, klare Service-Liste, häufige Bewertungen-Pflege | `hairdresser` / `barbershop` |
| **Auto-Werkstatt**    | Konkrete Leistungen, hoher Anfragewert, B2C-Vertrauenssache | `auto_workshop` |
| **Reinigungsfirma**   | B2B + B2C-Mix, einfache Lead-Form, niedriger Foto-Aufwand | `cleaning_company` |

Andere 10 Presets bleiben im Code, sind aber im Marketing als
„Coming Soon" markiert (ab Session 91). Erst nach 3 echten
Pilotkunden pro Hero-Branche kommen weitere Branchen aktiv
zurück.

## 4) Erstgespräch (Discovery, 30 Min)

**Ablauf**:
1. Begrüßung + Vorstellung (3 Min)
2. Status-Quo des Betriebs (8 Min)
   - Habt ihr schon eine Website? Wie zufrieden seid ihr?
   - Wie kommen aktuell Anfragen rein?
   - Wie sammelt ihr Bewertungen?
   - Welche Branche, wie viele Mitarbeiter?
3. Demo (10 Min)
   - Live-Demo einer der 3 Hero-Branchen
   - Anfrageformular ausfüllen lassen
   - Bewertungs-Booster zeigen
4. Pakete + Pilotkonditionen (5 Min)
5. Nächste Schritte (4 Min)
   - Vertrag-Versand binnen 24 h
   - 7-Tage-Onboarding-Slot reservieren

## 5) 7-Tage-Onboarding-Playbook

### Tag 0 — Briefing (30 Min, telefonisch oder Vor-Ort)

**Wir holen ein**:
- Betriebsname, Slogan/Tagline, Beschreibung (3 Sätze)
- Adresse, Telefon, E-Mail, evtl. WhatsApp-Nummer
- Logo (PNG/SVG)
- 3–8 Fotos (Innen, Team, Arbeit)
- Liste der Services mit Preis-Hinweis
- Öffnungszeiten
- Branche + bevorzugtes Theme

**Wir geben mit**:
- Inhalts-Checklist als PDF
- Sammlung-Link für Fotos (Cloud)
- Slot-Kalender für Tag 7

### Tag 1 — Setup

- Betrieb in Dashboard angelegt
- Branchen-Preset gewählt → 80 % der Inhalte automatisch vorbefüllt
- Theme zugewiesen
- Slug + Domain-Skizze

### Tag 2 — Inhalte

- Hero-Text final
- 3–10 Leistungen eingegeben (mit Preisen, Dauern, Beschreibungen)
- FAQ aus Branchen-Preset adaptiert (5–8 Fragen)
- Beispiel-Reviews eingegeben (mit Kunde-Erlaubnis)

### Tag 3 — Bilder

- Logo hochgeladen
- 1 Cover-Bild
- Pro Service ein Bild (UUID-gegated, S58-Pattern)
- Kompression + WebP automatisch

### Tag 4 — Tests

- Anfrageformular live testen → kommt bei Owner als E-Mail an
- Bewertungs-Booster mit echtem Kunde testen (1–2 Anfragen)
- Theme auf Mobile + Desktop validieren

### Tag 5 — Owner-Training

- 30 Min Video-Call: alle Editoren durchklicken
- Gemeinsam ein Service ändern, ein Bild tauschen, ein FAQ ergänzen
- Owner-Fragen sammeln + dokumentieren

### Tag 6 — Domain + Legal

- Domain registriert oder verbunden
- SSL aktiviert
- Impressum + Datenschutz mit echten Daten ausgefüllt
  (Owner muss prüfen + freigeben)
- Google Search Console eingerichtet

### Tag 7 — Live + Übergabe

- Site geht live
- 45-Min-Übergabe-Termin:
  - Login-Daten transferiert
  - Backup-Datenexport gezeigt
  - Support-Inbox + SLA erklärt
  - Kündigungs-/Datenexport-Pfad gezeigt
- Pilotkunde-Feedback-Form ausgefüllt

## 5b) Domain + Email-Setup (operativ, S87)

Volle Schritt-für-Schritt-Anleitung in
[`DOMAIN_SETUP.md`](./DOMAIN_SETUP.md). Empfohlene Konfig:
- Domain bei INWX (DE) oder Cloudflare-Registrar
- Vercel-Custom-Domain (Apex + WWW-Subdomain mit Redirect)
- Email-Hosting bei **mailbox.org** (DSGVO-konform, deutscher
  Anbieter, 3 €/Monat) — Vertrauenssignal für DE-Pilotkunden
- Postfach `kontakt@<domain>` + Aliase
  (`hello@`, `support@`, `dpa@`)
- SPF + DKIM + DMARC vollständig konfiguriert

Nach Durchführung: `docs/PRODUCT_STATUS.md` Sektion „Domain"
auf 4/4 ✅.

## 7) Customer-Support-Workflow (final ab Session 97)

### Standard-SLA
- Antwort binnen 1 Werktag
- Kritische Bugs (Site offline, Anfragen kommen nicht an):
  binnen 4 h Reaktion, 24 h Fix-Versuch
- Feature-Requests: gesammelt, monatlicher Status-Mail an Kunden

### Email-Templates
- Standard-Antrwort
- Bug-Report-Eingangsbestätigung
- Feature-Request-Status
- Kündigungs-Bestätigung + Datenexport-Versand
- Monatlicher Newsletter mit neuen Features

## 8) Übergabe-Checklist

```
[ ] Login-Daten an Owner übergeben (Magic-Link aktiviert)
[ ] Custom Domain live + SSL grün
[ ] Anfrageformular getestet (Test-Mail kommt an)
[ ] Mind. 1 echte Bewertung über Bewertungs-Booster eingeholt
[ ] Impressum + Datenschutz mit echten Daten validiert
[ ] Google Search Console verifiziert
[ ] Backup/Export-Pfad dem Owner gezeigt
[ ] Support-Email + SLA dem Owner schriftlich übergeben
[ ] Erste Rechnung verschickt
[ ] Kunden-Feedback-Form 30-Tage-Reminder gesetzt
```

## 9) 30-Tage-Feedback (Case-Study-Quelle)

Nach 30 Tagen Live-Betrieb:
- Wie viele Anfragen über das Formular?
- Wie viele neue Bewertungen?
- Welche Editoren nutzt der Owner aktiv?
- Was nervt am System?
- Wäre der Owner bereit, eine Case-Study zu erlauben?

→ Antworten füllen `docs/CASE_STUDIES/<branche>-<betrieb>.md`
(Pattern ab Session 93).
